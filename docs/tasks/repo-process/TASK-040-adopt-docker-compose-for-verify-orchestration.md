## Summary

Replace the low-level Docker orchestration inside `./mvnw clean verify` with Docker Compose while preserving Maven as the single verification entrypoint, relying on Compose service naming for inter-container database connectivity, requiring DB verification credentials from environment variables instead of repository-stored defaults, and removing the unnecessary database-selection Spring profile.

## Scope

- Add a dedicated Docker Compose file for local verification.
- Start PostgreSQL and the application container through Docker Compose in the Maven verification lifecycle.
- Keep the tested image tag generated by Maven and inject it into the Compose-based verification environment.
- Use the Compose `db` service name directly for the application-to-database JDBC hostname instead of passing a separate full JDBC URL variable into Compose.
- Read verification database username and password from environment variables instead of hard-coded repository defaults.
- Document `VERIFY_DB_USERNAME` and `VERIFY_DB_PASSWORD` as required prerequisites in the operator workflow before `./mvnw clean verify`.
- Remove the `postgres` Spring profile where it only exists to select the single supported database.
- Keep Playwright `@Tag("e2e")` and PostgreSQL-backed `@Tag("db")` tests running from Failsafe.
- Keep post-test cleanup and log capture aligned with the Compose workflow.

## Out of Scope

- Replacing Maven with Docker Compose as the main developer entrypoint.
- Publishing images to a remote registry during verification.
- Changing the OCI release/deploy workflow.
- Changing user-facing application behavior.

## Acceptance Criteria

- [x] `./mvnw clean verify` still remains the single required verification command.
- [x] Workflow docs explicitly require `VERIFY_DB_USERNAME` and `VERIFY_DB_PASSWORD` to be set before `./mvnw clean verify`.
- [x] Verification environment orchestration uses Docker Compose instead of multiple low-level `docker run` and `docker network` steps.
- [x] Maven still builds the fresh image first and verifies that exact image tag.
- [x] Compose service naming is used directly for application-to-database connectivity, without a redundant Compose JDBC URL variable.
- [x] Verification DB username/password are supplied from environment variables rather than repository-stored defaults.
- [x] The single supported PostgreSQL datasource no longer depends on a dedicated Spring profile.
- [x] `@Tag("db")` and `@Tag("e2e")` coverage still runs during `verify`.
- [x] Verification cleanup removes the Compose environment and preserves useful logs on failure.
- [x] `./mvnw clean verify` passes.
